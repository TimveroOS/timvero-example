= Password Management

timveroOS provides built-in password security for the back-office user authentication system.
Configure account lockout, password expiration, and password validation rules through the `password.*` properties in your `application.properties` file.

== Account Lockout

The platform locks a user account after too many consecutive failed login attempts within a rolling time window.
With the defaults below, 3 failures inside any 30-minute window triggers a lockout.
The lock is automatically lifted once the oldest recorded attempt falls outside the window.

[source,properties]
----
# Maximum number of failed attempts within the window before the account is locked
password.attempts=3

# Number of minutes a failed attempt remains counted toward the lockout threshold
password.block.minutes=30
----

NOTE: Failed-attempt counters are held in memory and are reset if the application restarts.

TIP: Increase `password.attempts` or decrease `password.block.minutes` to reduce friction for legitimate users. For stricter security, do the opposite.

When the number of remaining attempts drops to 3 or fewer, the login form shows the exact count.
An administrator can unlock an account at any time, which clears all recorded failed attempts immediately.

== Password Expiration

Passwords expire after a configurable number of days, counted from the moment the password was last set.
Once expired, the user is prompted to change their password on the next login.

[source,properties]
----
# Number of days before a password expires and must be changed
password.lifetime.days=90
----

NOTE: A password flagged as single-use (e.g. a temporary reset password) is treated as expired immediately, regardless of this setting.

== Password Change Flow

When a user's password has expired, the login page displays a change-password form.
The new password must differ from the current one and satisfy the `password.pattern` regex.
After a successful change the user is redirected to the login page to authenticate with the new password.

== Temporary Passwords

A 12-character random password is generated when a new user is created or when an administrator resets a user's password.
It is marked as single-use, so the user must change it on their very first login (see <<Password Expiration>>).
A notification containing the temporary password is sent automatically.

== Configuration Reference

[cols="3,1,5"]
|===
|Property |Default |Description

|`password.attempts`
|`3`
|Maximum number of failed login attempts allowed within the window before the account is locked.

|`password.block.minutes`
|`30`
|Length of the rolling window (in minutes) used to count failed login attempts.

|`password.lifetime.days`
|`90`
|Number of days after which a password expires and must be changed.

|`password.pattern`
|`.*`
|Java regex pattern that a new password must match. Use `.*` to allow any value (no restriction).
|===
